# Spec 002: 论文信息与研究方向标签管理

> 创建日期: 2026-05-25
> 状态: 已完成
> 对应 plan: [docs/plans/002-paper-and-tags-plan.md](../plans/002-paper-and-tags-plan.md)

## 目标

让系统能够沉淀论文元数据、两级研究方向标签、论文检索入口和论文下载入口，为后续用户行为记录、语义向量推荐和热门论文分析提供真实数据基础。

本 spec 完成后，登录用户可以提交论文，普通用户可以检索和查看论文，管理员可以维护论文、标签和软删除恢复状态，系统具备约 100 条样例论文数据用于展示和后续推荐链路验证。

## 用户故事

1. 作为普通用户，我想要提交论文基础信息，以便把自己关注的论文纳入系统。
2. 作为普通用户，我想要按标题、作者、年份、来源、研究方向标签和摘要关键词搜索论文，以便快速找到相关文献。
3. 作为普通用户，我想要查看论文详情和下载入口，以便阅读或跳转到原始论文资源。
4. 作为管理员，我想要维护固定研究方向标签，以便系统中的论文分类保持稳定。
5. 作为管理员，我想要编辑、软删除和恢复论文，以便纠正错误数据且避免误删。
6. 作为后续推荐系统，我想要论文具备标题、摘要、关键词和研究方向标签，以便生成 embedding 和相似度推荐。

## 功能需求

- [x] 新增论文主表，保存标题、作者、摘要、年份、来源、DOI、URL、下载链接、关键词、引用量、发布时间等字段。
- [x] 论文业务字段允许为空；系统字段如 id、状态、创建时间、更新时间必须完整。
- [x] 支持论文状态，至少包含正常、软删除两类；软删除后的论文可由管理员恢复。
- [x] 记录论文提交人，登录用户可以提交论文。
- [x] 普通用户提交的论文默认进入可见数据集，第一版不做复杂审核流。
- [x] 管理员可以新增、编辑、软删除、恢复论文。
- [x] 普通用户可以查看正常状态论文列表和论文详情。
- [x] 普通用户不能看到软删除论文，管理员可以在管理入口查看和恢复软删除论文。
- [x] 新增固定研究方向标签库，支持两级标签: 一级方向和二级方向。
- [x] 一篇论文可以绑定多个二级标签；二级标签必须归属某个一级标签。
- [x] 管理员可以维护标签启用状态；第一版标签不做无限层级。
- [x] 论文支持按标题关键词、作者、年份、来源、研究方向标签、摘要关键词搜索。
- [x] 搜索条件均为可选，多个条件组合时取交集。
- [x] 论文列表支持基础分页，避免一次返回全部数据。
- [x] 论文详情展示完整元数据、关键词、两级研究方向标签、URL 和下载入口。
- [x] 下载入口优先尝试真实下载论文文件。
- [x] 真实下载失败时，系统应保留外部下载链接并给出明确失败原因，不影响论文详情查看。
- [x] 下载结果需要记录文件名、文件大小、下载状态或失败原因，为后续行为记录预留入口。
- [x] 初始化或导入约 100 条样例论文数据，尽量覆盖多个研究方向标签。
- [x] 样例数据来源优先选择公开、无需登录、可稳定访问的论文元数据来源。
- [x] 前端提供论文列表、论文详情、论文提交、管理员论文维护、标签筛选入口。
- [x] 前端页面继续复用 Vue/Vite + Element Plus，保持简洁展示，不引入复杂状态管理。
- [x] Java 服务继续作为唯一前端 API 入口；前端不直接访问 MySQL、Redis 或 Python 服务。

## 边界条件

- **所有论文业务字段为空**: 允许提交，但列表和详情需要显示占位文案，不能前端崩溃。
- **标题过长**: 后端返回明确参数错误，前端显示错误提示。
- **摘要过长**: 后端返回明确参数错误，避免超出数据库字段或页面异常。
- **年份为空**: 允许保存，搜索年份为空时不参与过滤。
- **年份非法**: 返回明确参数错误，例如非数字或明显超出合理范围。
- **引用量为空**: 允许保存。
- **引用量为负数**: 返回明确参数错误。
- **DOI 重复**: 如果 DOI 非空，应避免重复入库或返回重复提示。
- **URL 或下载链接格式非法**: 允许保存为普通文本还是拒绝，需要在 plan 阶段按实现成本确定；至少不能导致下载任务崩溃。
- **绑定不存在的标签**: 拒绝保存，提示标签不存在或已禁用。
- **绑定一级标签**: 不允许直接绑定一级标签，论文只绑定二级标签。
- **搜索条件为空**: 返回正常论文分页列表。
- **搜索结果为空**: 返回空列表和总数 0，不报错。
- **普通用户编辑他人论文**: 第一版不允许，返回无权限。
- **普通用户访问软删除论文**: 返回不存在或无权限。
- **管理员恢复不存在的论文**: 返回明确错误。
- **重复软删除**: 幂等处理或返回当前已删除状态，不造成异常。
- **真实下载超时**: 标记下载失败并记录原因，详情页仍可展示外部链接。
- **下载内容不是 PDF**: 记录失败或非 PDF 状态，不能当作有效论文文件。
- **样例数据爬取失败**: 允许使用本地种子数据补足，但必须在验收记录里说明失败来源和补足方式。
- **Python 推理服务未启动**: 不影响本 spec 的论文管理、标签管理和搜索功能。

## 不在范围内

- 不做论文全文解析、PDF 内容抽取或全文检索。
- 不做复杂爬虫平台、反爬绕过、登录态抓取或多站点适配器体系。
- 不做论文审核流、多人协作编辑、版本历史或冲突合并。
- 不做复杂标签本体、无限层级标签或标签自动生成。
- 不做用户收藏、评分、浏览、点击、下载行为统计；这些放到 Spec 003。
- 不做 embedding 生成、相似度计算和个性化推荐；这些放到 Spec 004 和 Spec 005。
- 不做管理员完整管理台，只做本 spec 必需的维护入口。
- 不把下载到的论文文件提交进 git 仓库。

- [x] 登录用户可以提交一篇字段部分为空的论文，提交后能在列表中看到。
- [x] 管理员可以编辑论文元数据。
- [x] 管理员可以软删除论文，普通用户列表和详情不可见。
- [x] 管理员可以恢复软删除论文，恢复后普通用户可见。
- [x] 系统内存在固定两级研究方向标签，论文可以绑定多个二级标签。
- [x] 按标题、作者、年份、来源、研究方向标签、摘要关键词搜索均可得到正确结果。
- [x] 多条件组合搜索按交集过滤。
- [x] 论文列表分页可用。
- [x] 论文详情展示标题、作者、摘要、年份、来源、DOI、URL、下载链接、关键词、引用量、发布时间和研究方向标签。
- [x] 至少一次真实下载尝试能够成功保存下载状态和文件元数据。
- [x] 下载失败场景有明确状态和失败原因，页面仍能打开外部链接。
- [x] 初始化或导入约 100 条样例论文，且覆盖多个一级和二级研究方向标签。

- [x] Java 相关测试通过；前端构建通过。
- [x] README 或模块文档补充本 spec 的初始化、样例数据导入和验证方式。

## 已决策

- 论文字段包含标题、作者、摘要、年份、来源、DOI、URL、下载链接、关键词、引用量、发布时间。
- 上述论文业务字段第一版允许为空。
- 登录用户可以提交论文。
- 论文删除采用可恢复的软删除。
- 研究方向标签采用固定标签库。
- 标签层级固定为两级。
- 一篇论文可以绑定多个标签。
- 搜索支持标题、作者、年份、来源、研究方向标签、摘要关键词。
- 下载入口第一版尝试真实下载，并保留失败降级。
- 需要约 100 条样例论文数据。
- 前端需要论文列表、详情、提交、管理员维护和标签筛选入口。
- 样例论文优先使用 arXiv 公开 API 元数据。
- 下载文件保存到 git 仓库之外的本机目录，仓库只记录下载状态和本机路径。
- 普通用户提交论文后第一版立即公开。
- DOI 非空时按全局唯一处理，避免重复论文。
