# A1 项目定义

> 创建日期: 2026-05-24
> 路径: Vibe Coding V3 / 路径 A / 阶段一

## 1. 这个项目解决什么问题?

为科研文献管理场景提供论文管理、用户科研行为沉淀、论文相似度计算和个性化推荐能力，帮助用户更快发现与自己研究方向相关的论文。

## 2. 谁会用? 什么场景用?

- 科研学习者: 收藏、评分、整理论文，并获得相关论文推荐。
- 研究人员或学生: 围绕研究方向维护标签，查看相似论文和热门论文。
- 系统管理员或维护者: 管理论文元数据、观察用户行为和推荐结果质量。

典型场景:
- 用户阅读论文后收藏或评分，系统沉淀偏好。
- 用户进入首页或推荐页，系统返回热门论文、相似论文和个性化推荐。
- 维护者批量补充论文摘要、关键词、研究方向标签。

## 3. MVP 是什么?

MVP 控制在 5 个能力内:

1. 论文基础信息与研究方向标签管理。
2. 用户收藏、评分、访问行为记录。
3. Python 推理服务生成论文语义向量和相似度分数。
4. Java 推荐接口聚合推荐分数、相似论文和用户偏好标签。
5. Redis 缓存热门论文、推荐结果和用户偏好标签，并提供简洁前端展示。

## 4. 明确不做什么?

- 第一版不做复杂前端，只做简洁美观的展示页面，优先复用成熟组件。
- 第一版不做真实大规模分布式训练，只做轻量但真实的 embedding 相似度推荐模型和可替换模型接口。
- 第一版不做在线学习或实时模型训练，用户行为先作为推荐输入数据沉淀。
- 第一版不做多租户、机构权限、论文版权全文解析。
- 第一版不做生产级监控告警，只保留日志和后续扩展口子。

## 5. 预期活多久?

按“几个月项目”处理: 流程完整执行，但实现范围以 MVP 为主，避免一开始铺太大。

## 已确认

- 需要登录认证。
- 需要简洁前端页面，达到展示效果即可，不堆复杂代码。
- 用户行为尽量多记录，MVP 先覆盖浏览、搜索、点击、收藏、评分、下载入口、推荐点击等可沉淀行为。
- 论文数据支持爬虫实时更新，同时提供搜索/下载入口。
- 推荐模型使用真实、简单的 embedding 对比推荐模型，先做相似度推荐闭环。
- 科研行为分析优先展示热门论文和用户偏好标签。
- 未来扩展方向包括批量导入、模型评估、推荐解释、权限系统、前端管理台和论文全文解析。

## 待确认

- 推荐入口暂定为首页推荐、论文详情相似论文、搜索/下载入口推荐、个人推荐页，后续可调整。
- 爬虫第一版是否直接抓取公开论文站点，还是先通过可配置搜索链接和下载入口降低实现风险。
- 前端组件库最终选择，暂定 Vue/Vite + Element Plus。
